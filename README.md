# PMA Web

Sitio corporativo y comercial para **PMA Chile**, marca enfocada en accesorios y soluciones para **4x4, camionetas pickup y operaciones mineras**. La dirección del proyecto no es la de un ecommerce genérico: se construyó para comunicar **robustez, confianza, claridad comercial y compatibilidad real en terreno**, con una experiencia clara tanto para clientes **B2B** como **B2C especializado**.

## Posicionamiento de marca PMA

PMA se presenta como una marca de perfil **industrial, técnico y funcional**. Su valor no está en el marketing aspiracional, sino en resolver necesidades reales de operación:

- **Uso en terreno real**: polvo, ripio, vibración, carga, jornadas extensas y mantenimiento operativo.
- **Compatibilidad y criterio técnico**: productos asociados a plataformas vehiculares y contextos de uso concretos.
- **Robustez como atributo central**: materiales, fijaciones, modularidad y durabilidad tienen prioridad sobre la estética decorativa.
- **Relación con industrias exigentes en Chile**: especialmente minería, logística, contratistas, mantenimiento y usuarios 4x4 con requerimientos serios.

En términos visuales y narrativos, PMA debe percibirse como una marca que entiende la realidad operativa del mercado chileno, no como una tienda masiva de accesorios universales.

## Clientes objetivo

### 1. Empresas y operaciones B2B
Público con procesos de compra más racionales y técnicos, donde importan la continuidad operacional, la compatibilidad y la escalabilidad:

- Empresas mineras y contratistas mineros.
- Flotas logísticas y de mantenimiento en terreno.
- Áreas de abastecimiento, mantención y operaciones.
- Empresas que necesitan cotizar accesorios para múltiples vehículos o implementaciones por etapa.

### 2. Usuarios especializados B2C
Clientes individuales, pero con un perfil técnico o de uso exigente:

- Propietarios de camionetas pickup para trabajo.
- Usuarios 4x4 enfocados en terreno, rescate o expedición técnica.
- Clientes que priorizan resistencia, funcionalidad e integración con otros accesorios.

### 3. Mercado chileno
La comunicación está orientada a Chile, por lo tanto considera:

- Referencias de uso en minería y faena.
- Lenguaje profesional en español chileno neutral.
- Escenarios de operación en Santiago, norte minero y cobertura nacional.

## Principios de diseño

El diseño del sitio debe mantener una línea coherente con el posicionamiento de PMA.

### Industrial y limpio
- Uso de paleta oscura con acentos cálidos tipo cobre/arena para transmitir materialidad y robustez.
- Tarjetas modulares, bordes técnicos y separación clara de bloques de información.
- Jerarquía visual fuerte, con titulares precisos y datos fáciles de escanear.

### Funcional, no aspiracional
- Evitar clichés de lifestyle, aventura vacía o “startup branding”.
- El sitio debe comunicar operación, ingeniería práctica y continuidad de servicio.
- Cada sección debe ayudar a evaluar productos, compatibilidad o contacto comercial.

### No genérico ecommerce
- La navegación está planteada como sitio comercial-industrial, no como tienda masiva.
- La estructura enfatiza líneas de producto, ficha destacada, cotización para empresas y canales de contacto.
- El objetivo visual es parecer una solución confiable para abastecimiento y usuarios exigentes.

### Responsive y modular
- El layout se adapta a escritorio, tablet y móvil sin perder legibilidad.
- Los componentes son reutilizables para que nuevas categorías, idiomas o páginas se integren sin reescribir la base.

## Tono de voz

El contenido debe mantener un tono:

- **Técnico**: explica capacidades, materiales, compatibilidad y escenarios de uso.
- **Práctico**: evita adornos innecesarios y va directo a la utilidad del producto.
- **Industrial**: suena cercano a operaciones, flotas, faenas y equipamiento serio.
- **Sin marketing lifestyle**: no usar mensajes aspiracionales genéricos, slogans vacíos ni tono de marca “cool startup”.

### Ejemplos del tono correcto
- “Parachoques frontal de acero reforzado con puntos de izaje y preparación para winche.”
- “Cotización para flotas, contratistas y operaciones en terreno.”
- “Compatibilidad operacional por plataforma vehicular y contexto de uso.”

### Ejemplos a evitar
- “Lleva tu aventura al siguiente nivel.”
- “Diseñado para espíritus libres.”
- “La experiencia premium que tu estilo merece.”

## Filosofía de producto

Los productos PMA deben comunicarse bajo una lógica de uso real:

- **Durabilidad**: diseñados para ambientes demandantes y ciclos operativos intensivos.
- **Compatibilidad**: soluciones asociadas a modelos de pickup, sistemas auxiliares y necesidades de integración.
- **Uso verificable**: foco en rescate, carga, iluminación, seguridad y soporte estructural.
- **Escalabilidad**: una misma línea debe poder servir tanto a un cliente individual como a una flota completa.

La narrativa del producto debe responder a preguntas concretas:

- ¿Para qué escenario sirve?
- ¿Con qué plataformas o usos es compatible?
- ¿Qué problema operacional resuelve?
- ¿Qué tan fácil es integrarlo a una operación mayor?

## Objetivos del sitio web

Este proyecto fue estructurado para cumplir los siguientes objetivos:

1. **Conversión comercial**: facilitar contacto y solicitudes de evaluación o cotización.
2. **Soporte B2B + B2C**: servir tanto a empresas como a usuarios especializados.
3. **Claridad de productos**: mostrar líneas de producto y una ficha técnica destacada sin depender de un ecommerce completo.
4. **Posicionamiento de marca**: establecer a PMA como una marca seria, técnica e industrial para el mercado chileno.
5. **Base escalable**: dejar preparada la arquitectura para crecimiento posterior.

## Secciones implementadas

El sitio incluye las secciones solicitadas, ahora con un enfoque más directo a conversión y soporte comercial:

- **Home / Hero comercial**: propuesta de valor fuerte, CTA de cotización y estética industrial premium.
- **Confianza**: mensajes rápidos de respaldo comercial y operacional.
- **Products**: cards visuales con beneficio claro y llamado a revisar detalle.
- **Product detail**: bloque que explica cómo PMA estructura una solución operacional.
- **Companies**: sección B2B para flotas, contratistas y operaciones.
- **Differentiation**: franja de mensajes que refuerza el posicionamiento industrial.
- **Contact**: formulario corto, canales visibles y WhatsApp comercial.

## Gestión de assets visuales

Los archivos de marca, fotos, mockups y referencias visuales deben vivir en `media/`, separados por tipo de recurso para que el proyecto siga ordenado a medida que crezca. La estructura base ya considera logos, colores, tipografía, fotos de hero, fotos de producto y material visual de referencia.

- `media/branding/logos/original/`: originales del logo (por ejemplo `PMA-Logo.jpeg`).
- `media/branding/logos/exported/`: versiones listas para web o piezas comerciales.
- `media/branding/colors/`: paleta oficial y sistema de color.
- `media/photos/hero/`: fotos principales de portada.
- `media/photos/products/`: fotos por línea de producto.
- `media/photos/reference/`: referencias visuales o fotos de inspiración.
- `media/visuals/mockups/`: propuestas de Canva/Figma y capturas de diseño.

## Explicación de la arquitectura

### `src/content/siteData.js`
Contiene el contenido y la configuración de idiomas.

- `locales` define los idiomas disponibles.
- `content.es` contiene la versión principal en español.
- `content.en` deja la base preparada para una futura versión secundaria en inglés.
- Los productos, navegación y textos viven fuera de los componentes para facilitar mantenimiento, escalabilidad y traducción.

### `src/components/`
Componentes reutilizables para bloques repetidos:

- `SectionHeading`: encabezados consistentes para secciones.
- `ProductCard`: card de producto enfocada en beneficio comercial y CTA.
- `LanguageBadge`: indicador visual de arquitectura bilingüe.
- `LogoMark`: adaptación vectorial del isotipo para branding consistente en navbar, hero y fallback.
- `IconBadge`: set simple de íconos industriales para confianza, contacto y productos.
- `ErrorBoundary`: evita una pantalla completamente en blanco si ocurre un error de render en cliente.

### `src/layouts/SiteLayout.jsx`
Layout principal del sitio con:

- encabezado sticky,
- navegación por anclas,
- estructura de página,
- footer de posicionamiento.

### `index.html`
Incluye un shell visual mínimo de respaldo para que el usuario vea contenido de marca incluso si la carga de JavaScript falla o se demora.

### `src/pages/HomePage.jsx`
Compone todas las secciones del sitio actual en una sola página larga. Esto acelera la validación comercial inicial y permite migrar luego a rutas reales si el proyecto evoluciona.

### `src/styles/index.css`
Centraliza:

- estilos base,
- utilidades de diseño,
- clases reutilizables (`section-shell`, `panel`, `button-primary`, etc.),
- identidad visual industrial.

## Soporte multiidioma futuro

Aunque el sitio actual está construido con **español como idioma principal**, la estructura ya considera crecimiento hacia una segunda capa en inglés.

### Decisiones tomadas para soportar múltiples idiomas
- Separación entre contenido y presentación.
- Objeto `locales` para administrar idiomas disponibles.
- Objeto `content` con clave por idioma.
- Configuración dinámica de `document.title`, `meta description` y `lang` del documento.

### Siguiente paso recomendado
Si el proyecto necesita navegación real por idioma, se puede evolucionar hacia:

- `react-router-dom` con rutas `/es` y `/en`, o
- una librería como `i18next` si el contenido crece mucho.

## Cómo ejecutar el proyecto localmente

### Requisitos
- Node.js 18+ recomendado.
- npm 9+ recomendado.

### Instalación
```bash
npm install
```

### Desarrollo local
```bash
npm run dev
```

Luego abrir en el navegador la URL local que entregue Vite (normalmente `http://localhost:5173`).

### Build de producción
```bash
npm run build
```

### Vista previa del build
```bash
npm run preview
```

## Escalabilidad futura

La base actual está pensada para crecer sin romper el enfoque de marca.

### 1. Ecommerce controlado
Se puede añadir:
- catálogo con filtros por marca, modelo y uso,
- stock o disponibilidad referencial,
- carrito o solicitud híbrida de compra/cotización,
- integración con pasarelas o ERP.

### 2. B2B avanzado
Se puede incorporar:
- formularios con archivos adjuntos,
- cotización por flota y proyecto,
- cuentas corporativas,
- historial de requerimientos,
- fichas técnicas descargables,
- gestión por rubro o tipo de operación.

### 3. Catálogo multiidioma
La separación de contenido permite:
- desplegar versión en inglés para proveedores o clientes internacionales,
- mantener contenido técnico consistente por idioma,
- traducir productos sin rehacer componentes.

### 4. Evolución de navegación
La página única actual puede migrarse a:
- rutas dedicadas por sección,
- páginas por categoría,
- páginas por producto,
- micrositios por industria (minería, logística, rescate, 4x4 profesional).

## Dirección recomendada para futuros desarrolladores o asistentes de IA

Cualquier evolución del proyecto debe preservar estas reglas:

1. **No convertir la marca en ecommerce genérico** sin justificación estratégica.
2. **Mantener el tono técnico e industrial** en todos los textos.
3. **Priorizar compatibilidad, uso real y claridad operacional** sobre claims de lifestyle.
4. **Diseñar para Chile primero**, especialmente para minería, logística y pickups de trabajo.
5. **Escalar desde una base modular**, con contenido desacoplado y componentes reutilizables.

Si una nueva funcionalidad entra en conflicto con estas reglas, debe adaptarse a la identidad PMA antes de implementarse.
